# 사업계획서

## 엣지 세이버 (Edge Saver) — 엣지 AI 기반 지능형 화재 감시 시스템

---

## 1. 사업 개요

### 1.1 사업명
**엣지 세이버 (Edge Saver)** — 멀티센서 + 엣지 AI 기반 지능형 화재 감시 및 대응 시스템

### 1.2 사업 목적
기존 화재경보기의 높은 오경보율과 단순 알림 한계를 극복하는 차세대 화재 감시 솔루션 개발. 멀티센서(연기·가스·온도)와 카메라 AI를 결합하여 **화재를 정확히 판별**하고, LLM+RAG 기술로 **건물 맞춤 대응 지침을 자동 생성**하여 인명 피해를 최소화한다.

### 1.3 사업 비전
> "모든 건물에 AI가 지키는 화재 안전망을"

기존 화재경보기를 대체하는 것이 아닌, **기존 시스템 위에 AI 지능을 얹어** 화재 감시의 패러다임을 전환한다.

## 2. 시장 분석

### 2.1 시장 현황 및 규모

| 항목 | 수치 |
|------|------|
| 국내 화재 발생 건수 (2024) | 약 38,000건 |
| 화재로 인한 인명 피해 (2024) | 사망 약 300명, 부상 약 1,800명 |
| 오경보로 인한 출동 비율 | 전체 출동의 약 30~40% |
| 국내 소방 안전설비 시장 규모 | 약 3조 원 |
| 세계 스마트 화재감지 시장 (2030 전망) | 약 60조 원 (CAGR 8.5%) |

### 2.2 기존 시스템의 문제점

| 문제 | 상세 |
|------|------|
| **높은 오경보율** | 연기감지기 단독 사용 시 조리 연기·먼지·수증기에도 반응 → 경보 무시 습관화 |
| **정보 부재** | "화재 발생"이라는 단순 경보만 제공, 어디서·어떤 화재인지·무엇을 해야 하는지 알려주지 않음 |
| **클라우드 의존** | 최신 IoT 화재감지 시스템은 서버 의존 → 화재 시 통신 인프라 손상되면 무력화 |
| **고비용** | 기업용 지능형 화재감지 시스템은 구역당 수백만 원대 |

### 2.3 타겟 시장

| 우선순위 | 타겟 | 이유 |
|----------|------|------|
| 1순위 | 중소 제조공장·물류창고 | 화재 위험 높음, 기존 설비 노후화, 비용 민감 |
| 2순위 | 소규모 건물 관리업체 | 건물별 맞춤 관리 필요, 전문 인력 부족 |
| 3순위 | 건설 현장·임시 시설 | 임시 설비로 빠른 설치/철거 필요, 외국인 근로자 다국어 안내 수요 |

### 3.1 제품 구성

```
┌─────────────────────────────────────┐
│         엣지 세이버 노드 (1대)        │
│                                     │
│  📦 하드웨어                        │
│  ├─ Raspberry Pi 5 (8GB)           │
│  ├─ MQ-2 연기센서                   │
│  ├─ MQ-135 가스센서                 │
│  ├─ DHT22 온도/습도센서              │
│  ├─ CSI 카메라 모듈                  │
│  ├─ 부저 + LED 경고등               │
│  └─ 산업용 방진 케이스               │
│                                     │
│  💾 소프트웨어                       │
│  ├─ Vision AI (화재/연기 영상 판별)   │
│  ├─ 센서 퓨전 엔진 (위험도 Level 1~5)│
│  ├─ LLM + RAG (건물 맞춤 대응 지침)  │
│  ├─ 관제 대시보드 (웹 기반)          │
│  └─ TTS 다국어 음성 안내             │
└─────────────────────────────────────┘
```

### 3.2 핵심 기능 및 차별점

| # | 기능 | 기존 경보기 | 엣지 세이버 |
|---|------|-----------|-------------|
| 1 | **AI 오경보 필터링** | ❌ | 센서 반응 시 카메라 AI가 2차 검증 → 오경보 80% 감소 |
| 2 | **건물 맞춤 자연어 안내** | ❌ | "전기실 화재, CO2 소화기 사용, 서쪽 계단 대피" |
| 3 | **위험도 등급화** | ON/OFF만 | Level 1(주의) ~ Level 5(재난) 세분화 |
| 4 | **다국어 음성 안내** | 한국어만 | 한국어/영어/베트남어/중국어 실시간 변환 |
| 5 | **관제사 AI Q&A** | ❌ | "B동 가장 가까운 소화전 어디?" 자연어 질의 |
| 6 | **자동 사고 보고서** | ❌ | 센서 로그 + 영상 기반 자동 작성 |
| 7 | **완전 오프라인** | △ | 통신 다운 시에도 독립 작동 |
| 8 | **저비용** | - | 경쟁 제품 대비 1/10 비용 |

### 3.3 작동 프로세스

```
[평상시] 센서 모니터링 (24/7)
         │
         ▼ 이상 감지
[1단계] 카메라 자동 촬영 → Vision AI 분석
         │
         ├─ "화재 아님" → 오경보 차단 (로그 기록)
         │
         └─ "화재 확인"
              │
[2단계] 멀티센서 퓨전 → 위험도 Level 산정
              │
[3단계] RAG: 건물 DB 검색 (위험물, 소화기, 대피로)
              │
[4단계] LLM: 맞춤형 대응 지침 자연어 생성
              │
[5단계] 경보 발령 + 음성 안내 + 관제실 알림
```

## 9. 기대 효과

### 기술적 효과
- AI 영상 2차 검증으로 **오경보율 80% 이상 감소** 기대
- 화재 감지 → 맞춤 대응 지침 생성까지 **10초 이내** 응답
- 통신 장애 시에도 **오프라인 독립 작동** 보장

### 경제적 효과
- 기존 스마트 화재감지 대비 **1/10 비용**으로 도입 가능
- 오경보 감소로 인한 **불필요 소방 출동 비용 절감**
- AI 자동 보고서로 **사고 분석 인력/시간 절약**

### 사회적 효과
- **인명 피해 최소화**: 건물 맞춤 대피 안내로 골든타임 확보
- **외국인 근로자 보호**: 다국어 음성 안내로 언어 장벽 해소
- **소방 자원 효율화**: 오경보 감소 → 실제 화재에 자원 집중
