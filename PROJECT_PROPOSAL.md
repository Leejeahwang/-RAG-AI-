# 🔥 프로젝트 최종 계획서

## Edge Saver — 엣지 AI 기반 지능형 화재 감시 시스템

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
**엣지 세이버 (Edge Saver)** — 라즈베리파이 기반 지능형 화재 감시 및 대응 시스템

### 1.2 한 줄 요약
> 멀티센서(연기·가스·온도)와 카메라 AI를 결합하여, 기존 화재경보기가 할 수 없는 **"화재 상황을 이해하고 건물 맞춤 대응 지침을 자동 생성"**하는 엣지 AI 시스템

### 1.3 프로젝트 배경 및 필요성

**기존 화재경보기의 근본적 한계:**

| 문제점 | 설명 |
|--------|------|
| 높은 오경보율 | 조리 연기, 먼지, 담배 연기에도 경보가 울려 실제 위험 시 무시되는 "양치기 소년" 현상 발생 |
| 단순 ON/OFF 알림 | "화재 발생" 한마디뿐, 어디서·어떤 종류의 화재인지·무엇을 해야 하는지 알려주지 않음 |
| 건물 맞춤 대응 불가 | 전기실 화재에 물 소화기를 쓰면 감전 위험이 있지만, 기존 경보기는 이를 안내할 수 없음 |
| 네트워크 의존 (IoT형) | 클라우드 기반 스마트 화재감지 시스템은 화재로 통신 인프라가 다운되면 무용지물 |
| 사후 분석 불가 | 센서 로그나 영상 기록이 없어 사고 원인 분석이 어려움 |

**→ "기존 화재경보기 + AI"로 이 문제들을 근본적으로 해결하는 것이 본 프로젝트의 목표입니다.**

### 1.4 왜 라즈베리파이인가? (스마트폰/클라우드 대비 차별점)

| 비교 항목 | 스마트폰 | 클라우드 IoT | **라즈베리파이 (본 시스템)** |
|-----------|---------|-------------|---------------------------|
| 24/7 고정 설치 감시 | ❌ 불가 | ⭕ 가능 | ⭕ 가능 |
| 센서 하드웨어 직결 (GPIO) | ❌ 불가 | △ 별도 MCU 필요 | ⭕ **직접 연결** |
| 화재 시 오프라인 동작 | ❌ (LTE 의존) | ❌ (서버 의존) | ⭕ **완전 독립** |
| 대규모 배치 비용 (20개 기준) | 약 2,000만원 | 월 구독료 발생 | **약 200만원** |
| 산업 환경 내구성 | ❌ (고온/분진) | - | ⭕ (산업용 케이스) |
| AI 추론 (로컬) | △ (제한적) | ⭕ (서버) | ⭕ **엣지 AI** |

> **결론:** 센서 직결 + 오프라인 AI + 저비용 대량 배치가 동시에 가능한 플랫폼은 라즈베리파이뿐입니다.

### 2.2 데이터 흐름 (이벤트 발생 시)

```
센서 이상 감지 (연기/가스/온도)
       │
       ▼
카메라 자동 촬영 → Vision AI 분석
       │
       ├─ "화재 아님" → 오경보 차단 (로그만 기록)
       │
       └─ "화재 확인" → 위험도 등급 산정 (Level 1~5)
              │
              ▼

              │
              ▼
       LLM: 상황별 맞춤 대응 지침 자연어 생성
              │
              ├─→ 🔔 경보 발령 + 음성 안내
              ├─→ 📱 관제실 알림 전송
              └─→ 📝 사고 로그 자동 기록
```

## 3. 핵심 기능 상세

### 3.1 AI 기반 오경보 필터링 (False Alarm Filter)
**기존 문제:** 연기센서 단독으로는 조리 연기, 먼지, 수증기를 구분할 수 없어 오경보율이 매우 높음.

**해결 방법:**
1. 센서가 이상 수치를 감지하면 → 카메라가 자동으로 해당 구역을 촬영
2. Vision AI(경량 모델)가 촬영된 영상에서 **실제 불꽃/연기 패턴을 분석**
3. "실제 화재"로 판정된 경우에만 경보 발령 → **오경보 대폭 감소**

### 3.2 멀티센서 퓨전 기반 위험도 등급화

| Level | 조건 | 대응 |
|-------|------|------|
| 1 (주의) | 가스 농도 미세 상승 | 로그 기록, 관제실 알림 |
| 2 (경고) | 연기 or 온도 단일 센서 반응 | 카메라 확인 요청, 관제실 경고 |
| 3 (위험) | 복수 센서 반응 + 카메라 연기 확인 | 해당 구역 경보, 대피 안내 시작 |
| 4 (긴급) | 복수 센서 + 카메라 불꽃 확인 | 전관 경보, 소방서 자동 신고, 전체 대피 |
| 5 (재난) | 다수 구역 동시 반응 + 확산 감지 | 건물 전체 비상, 인접 노드 연쇄 경보 |

### 3.3 LLM + RAG: 건물 맞춤 자연어 대응 지침 생성

**RAG 데이터베이스에 적재할 문서:**
- 건물 도면 및 구역별 용도 정보
- 위험물질 보관 위치 및 종류
- 소화기·소화전·비상구 위치 데이터
- 소방청 화재 대응 매뉴얼
- 건물별 비상 대피 계획서

**LLM이 생성하는 알림 예시:**
```
[Level 4 긴급 - B동 3층 전기실]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ B동 3층 전기실(구역 B-3-E)에서 화재가 확인되었습니다.

🚫 주의: 해당 구역에 고압 전기 설비가 있습니다.
   → 물 소화기(ABC형) 사용을 금지합니다.
   → CO2 소화기를 사용하세요.

🧯 가장 가까운 CO2 소화기: 3층 복도 동쪽 끝 (약 12m)
🚪 권장 대피 경로: 서쪽 비상계단 → 1층 후문
   (동쪽 계단은 화재 구역과 인접하여 위험)

👥 3층 잔류 인원: 확인 중...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 3.4 다국어 음성 대피 안내
LLM이 생성한 대피 지침을 TTS를 통해 **한국어·영어·베트남어·중국어** 등 다국어로 음성 방송. 외국인 근로자가 많은 공장/건설 현장에서 특히 유용.

### 3.5 관제실 대화형 AI 어시스턴트
긴급 상황에서 관제사가 AI에게 자연어로 즉시 질문 가능:
- "B동에서 가장 가까운 소화전 위치 알려줘"
- "현재 3층에 위험물질 보관된 곳 있어?"
- "A동과 B동 사이 연결 통로 폐쇄해도 대피에 문제없어?"

### 3.6 자동 사고 보고서 생성
화재 진압 후, LLM이 전체 센서 로그·카메라 기록·타임라인을 분석하여 자동으로 사고 경위 보고서를 작성. 소방서 제출용 서류 작성 시간을 대폭 절감.

## 4. 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| **하드웨어** | Raspberry Pi 5 (8GB) | 메인 엣지 컴퓨팅 보드 |
| **센서** | MQ-2 (연기), MQ-135 (가스), DHT22 (온도/습도) | 환경 이상 감지 |
| **카메라** | CSI 카메라 모듈 또는 USB 웹캠 | 영상 촬영 및 AI 분석 |
| **Vision AI** | OpenCV + 경량 화재 감지 모델 | 불꽃/연기 영상 판별 |
| **LLM** | Ollama + Qwen 2.5 (1.5B) | 자연어 대응 지침 생성 |
| **RAG** | LangChain + ChromaDB | 건물 DB 검색 증강 생성 |
| **음성** | pyttsx3 (TTS) | 오프라인 음성 안내 출력 |
| **통신** | MQTT / 로컬 네트워크 | 관제실 알림 및 노드 간 통신 |
| **컨테이너** | Docker + Docker Compose | 서비스 패키징 및 배포 |

## 8. 기대 효과

1. **오경보율 감소:** AI 영상 2차 검증으로 기존 대비 오경보 80% 이상 감소 기대
2. **골든타임 확보:** 화재 감지 → 맞춤 대응 지침 생성까지 10초 이내
3. **인명 피해 최소화:** 건물 구조를 이해하는 맞춤 대피 경로 + 다국어 음성 안내
4. **비용 절감:** 라즈베리파이 기반으로 기존 스마트 화재감지 시스템 대비 1/10 비용
5. **사후 분석 효율화:** AI 자동 사고 보고서로 원인 분석 시간 단축

## 9. 향후 확장 가능성

- **다중 노드 메시 네트워크:** 건물 전체에 분산 배치하여 구역별 독립 감시 + 연쇄 경보
- **잔류 인원 감지:** 대피 중 위험 구역에 남아있는 사람 자동 감지 및 구조 요청
- **예측 정비:** 센서 열화 감지 및 점검 주기 자동 알림
- **건물 관리 시스템(BMS) 연동:** 자동 방화문 폐쇄, 스프링클러 작동 등 연계
- **소방서 직접 연동:** 화재 시 119에 자동 신고 + 현장 영상 실시간 전송
